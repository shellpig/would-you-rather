# 驗證後已知問題

本文件由 verifier 維護,記錄驗證後發現的阻擋問題、待修問題、已修復項目與已接受邊界。

## [Phase 1][已修復] 首次選擇後比例百分比不顯示

- **目前處理狀態**:已修復(implementer 自我驗收通過)。Phase 1/2 驗收阻擋已解除。

- **內容說明**:首次進入任一題並點選選項後,選中 / 淡化狀態與比例條伸長皆正常,比例數值也已正確
  寫入 DOM,但動畫結束後百分比文字仍維持隱藏(`opacity: 0`)。實測等待超過 0.6 秒 transition 時間後
  仍未出現;只有再次改選另一邊,走「已播過動畫」分支時才會顯示。問題位於
  `src/pages/quizFlow.js > updateRatioSide()`:目前完全依賴 `transitionend` 加上 `.show`,但元素剛從
  `hidden` 切為可見時,單層 `requestAnimationFrame` 未可靠建立 `0% → 目標寬度` 的 transition,
  導致部分瀏覽器不觸發事件。此問題違反 `測試指南.md > Phase 1` 第 7 條,並影響第 11 條及
  `Phase 2` 第 13 條回歸驗收。

- **修復方式**:`src/pages/quizFlow.js > updateRatioSide()`(第一次選擇分支)改用雙層
  `requestAnimationFrame` 可靠提交比例條初始 `width: 0%`(避免元素剛從 `hidden` 轉可見時,單層 rAF
  與「取消 hidden」這次樣式套用被瀏覽器合併成同一次繪製而不觸發 transition),第二層 rAF 才寫入
  目標寬度;同時加入 650ms(略長於 CSS `transition: width 0.6s`)的 `setTimeout` fallback,用 `done`
  旗標防止與 `transitionend` 重複觸發淡入。改選(已播過動畫)分支與其餘行為不變。

- **驗收結果**:375×812 手機視窗,dev server + 本機 Worker/D1 真 API 走查。首次點選（重整頁面、清
  localStorage 後從乾淨狀態)比例條動畫跑完後百分比正確淡入、兩邊加總 100 且為整數;demo 題庫 6 題
  逐題走完,每題首次點選百分比皆正常出現;改選瞬間切換、不重播動畫;續玩、D1 對票、`npm test`
  22 條自動化測試皆不退化。另以「rAF 完全不執行」(比 reduced-motion 更嚴苛的退化情境,實測環境
  背景分頁下 rAF 被瀏覽器節流暫停)驗證 timeout fallback 確實獨立於動畫機制運作,百分比仍正確顯示。
