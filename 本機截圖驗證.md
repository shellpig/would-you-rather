# 本機截圖驗證

用無頭 Chrome(puppeteer-core)在本機自動跑完測驗流程並截圖,做為上線前的畫面驗證手段。**全程不需要部署**,只要本機 dev server 有起來就能跑。

## 背景

Claude Code 瀏覽器面板的 `screenshot` 功能曾發生持續 timeout(頁面本身正常、`read_page` 可讀,僅截圖管線故障,2026-07-19 驗證確認)。本文件的做法完全繞過面板,直接驅動本機安裝的 Chrome,不受該故障影響;即使面板修好,此法也適合做可重複的自動化畫面驗收。

## 原理

- `puppeteer-core` 驅動本機已安裝的 Chrome(headless),**不會**下載額外瀏覽器。
- 腳本開 `http://localhost:5173`(Vite dev server),API 經 vite proxy 轉本機 8788 Worker,資料讀寫本機 dev D1 —— 從頭到尾不碰線上環境。
- 視口固定 375×812(mobile)、`deviceScaleFactor: 2` 輸出清晰 2x 圖。

## 前置需求

1. 本機已安裝 Chrome(預設路徑 `C:\Program Files\Google\Chrome\Application\chrome.exe`;沒有 Chrome 時可改用 Edge `C:\Program Files (x86)\Microsoft\Edge\Application\msedge.exe`,改腳本內 `CHROME` 常數即可)。
2. 兩個 dev server 已啟動(命令見 `測試指南.md > 測試環境`):
   - `npm run dev` → 5173(前端)
   - `npm run dev:worker` → 8788(Worker + dev D1)
3. 一次性安裝:在**專案外**任一工作資料夾(避免動到專案 `package.json`)執行:

   ```
   npm init -y
   npm install puppeteer-core
   ```

   然後把下方腳本存成該資料夾內的 `snap-summary.js`。

## 腳本(snap-summary.js)

```js
// 驗證用腳本:無頭 Chrome 跑完 daily-life 15 題,截總結卡。
const puppeteer = require("puppeteer-core");
const path = require("path");

const CHROME = "C:\\Program Files\\Google\\Chrome\\Application\\chrome.exe";
const BASE_URL = "http://localhost:5173"; // 驗 build 產物時改 8789(見下方「驗證環境」)
const QUIZ = "daily-life";
// 選邊路線(對應題序 1..N)。此路線只在第 1 題站少數派 → 觸發「毛孩孤勇者」稱號。
const SIDES = ["b", "a", "a", "b", "a", "a", "a", "a", "a", "b", "a", "b", "b", "b", "b"];
const OUT = __dirname;

(async () => {
  const browser = await puppeteer.launch({ executablePath: CHROME, headless: "new" });
  const page = await browser.newPage();
  await page.setViewport({ width: 375, height: 812, deviceScaleFactor: 2 });

  await page.goto(`${BASE_URL}/quiz/${QUIZ}`, { waitUntil: "load" });
  await page.evaluate(() => localStorage.clear()); // 清進度,確保從第 1 題開始
  await page.reload({ waitUntil: "load" });

  await page.waitForFunction(
    () => [...document.querySelectorAll("button")].some((b) => b.textContent.includes("開始作答")),
    { timeout: 10000 }
  );
  await page.screenshot({ path: path.join(OUT, "shot-intro.png") });

  await page.evaluate(() =>
    [...document.querySelectorAll("button")].find((b) => b.textContent.includes("開始作答")).click()
  );

  for (let i = 0; i < SIDES.length; i++) {
    const side = SIDES[i];
    // 新題渲染完成的訊號:下一題按鈕存在且 disabled
    await page.waitForFunction(() => {
      const b = document.querySelector("[data-next]");
      return b && b.disabled;
    }, { timeout: 10000 });
    await page.evaluate((s) => document.querySelector(`.option-card[data-side="${s}"]`).click(), side);
    await page.waitForFunction(() => {
      const b = document.querySelector("[data-next]");
      return b && !b.disabled;
    }, { timeout: 10000 });
    await new Promise((r) => setTimeout(r, 120));
    await page.evaluate(() => document.querySelector("[data-next]").click());
    console.log(`Q${i + 1} -> ${side}`);
  }

  await page.waitForSelector(".result-card", { timeout: 10000 });

  // 徽章圖載入驗證:src 與 naturalWidth(0 = 破圖)
  const badgeInfo = await page.evaluate(async () => {
    const img = document.querySelector(".result-badge__img");
    if (!img) return { exists: false };
    if (!img.complete) await new Promise((res) => { img.onload = res; img.onerror = res; });
    return { exists: true, src: img.src, naturalWidth: img.naturalWidth };
  });
  console.log("badge:", JSON.stringify(badgeInfo));
  console.log("badgeName:", await page.evaluate(
    () => document.querySelector(".result-badge__name")?.textContent ?? null
  ));

  await new Promise((r) => setTimeout(r, 500)); // 等動畫/版面靜定

  await (await page.$(".result-hero")).screenshot({ path: path.join(OUT, "shot-hero.png") });
  await (await page.$(".result-list")).screenshot({ path: path.join(OUT, "shot-list.png") });
  await page.screenshot({ path: path.join(OUT, "shot-full.png"), fullPage: true });

  await browser.close();
  console.log("DONE");
})().catch((e) => { console.error(e); process.exit(1); });
```

## 執行與輸出

```
node snap-summary.js
```

輸出(存在腳本同資料夾):

| 檔案 | 內容 |
|---|---|
| `shot-intro.png` | 測驗首頁(intro),兼截圖功能自檢 |
| `shot-hero.png` | 總結卡上半:勳章區(徽章圖 + 稱號三行字 + 比例句 + 判詞) |
| `shot-list.png` | 總結卡下半:兩欄逐題結算格 |
| `shot-full.png` | 總結卡整頁全圖 |

console 同時印出徽章圖 `src` 與 `naturalWidth`(0 表示破圖)、稱號名,可直接核對。

## 驗證環境

| 目標 | `BASE_URL` | 前置 | 用途 |
|---|---|---|---|
| dev(日常) | `http://localhost:5173` | dev + dev:worker 起著 | 改碼即時驗證 |
| build 產物(上線前建議) | `http://localhost:8789` | `npm run build` 後啟動 `preview:dist` | 驗「實際要部署的那份」 |
| 正式站 | — | — | **不要跑**:自動作答會對正式 D1 投真票 |

## 注意事項

- **每跑一次會對 dev D1 各題實投一票**(首次作答即送票),本機統計數字會被墊高;屬預期行為,不影響正式環境。
- 腳本開頭會 `localStorage.clear()`,無頭瀏覽器的 profile 是獨立的,不影響你日常瀏覽器裡的作答進度。
- 換題庫:改 `QUIZ` 與 `SIDES`(長度需等於該庫題數)。想驗特定稱號,依 `subdocs/題庫/<id>.md` 的種子票分佈設計選邊路線。
- 想驗其他畫面(首頁、答題中途),在對應時點加 `page.screenshot(...)` 即可;元素級截圖用 `(await page.$("selector")).screenshot(...)`。
